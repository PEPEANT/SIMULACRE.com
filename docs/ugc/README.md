# UGC

시뮬라크 시티의 UGC 기준을 모아두는 저장소다.
이 저장소는 실행 코드 저장소가 아니라, 오브젝트 배치와 유저 흔적, 포탈 구조, UGC 철학을 정리하는 기준 저장소다.

## 한 줄 정의

UGC는 "시뮬라크 안에서 유저와 AI가 남기는 흔적이 세계의 일부가 되도록 만드는 시스템의 근간"이다.

## 이 저장소가 하는 일

- 시뮬라크 UGC의 철학과 세계 기준을 보존한다.
- 오브젝트 배치의 의미와 사용 원칙을 기록한다.
- 포탈, 시작 구역, 본체 세계의 관계를 고정한다.
- AI 에이전트가 문서를 읽을 때 헷갈리지 않도록 기준 우선순위를 제공한다.

## 이 저장소가 하지 않는 일

- 실행 코드의 단일 소스가 되지 않는다.
- 미니게임 기획서 모음으로 축소되지 않는다.
- 임시 이벤트 문서를 본체 철학보다 우선하지 않는다.
- 장식용 설정집으로 끝나지 않는다.

## 문서 우선순위

에이전트와 사람 모두 아래 순서로 해석한다.

1. `AGENTS.md`
2. `ugc-rules.yaml`
3. `EXPORT_RULES.md`
4. `SIMULACRA_UGC_PLATFORM.md`
5. 이후 추가되는 세부 문서

상위 문서와 하위 문서가 충돌하면 상위 문서를 따른다.

해석 기준은 다음과 같다.

- `DIS\EM` 문서는 장기 비전과 세계관 기준을 담당한다.
- `DIS\EM`의 계약 문서는 연속성과 구현 목표를 담당한다.
- UGC 규칙이 장기 비전과 충돌하면, 표현을 보강해서 둘을 맞추고 함부로 덮어쓰지 않는다.

## 핵심 개념

- `시뮬라크`: 개별 콘텐츠를 담는 본체 세계
- `공`: 시뮬라크 내부의 0번 구역, 시작과 선택의 공간
- `포탈`: 교체 가능한 콘텐츠 모듈로 연결되는 관문
- `오브젝트`: 장식이 아니라 유저 흔적과 창작의 매개체
- `UGC`: 만들 수 있음에서 끝나지 않고, 세계 안에서 쓰임을 갖는 구조

## 읽는 이유

이 저장소는 "무엇을 만들 것인가"보다 먼저 "무엇을 절대 헷갈리면 안 되는가"를 정리하기 위해 존재한다.
따라서 문서를 수정할 때는 표현 추가보다 개념 경계 유지가 우선이다.
