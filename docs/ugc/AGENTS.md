# AGENTS.md

## 목적

이 저장소는 시뮬라크 시티의 UGC 기준 저장소다.
코드 구현 저장소로 오해하지 말고, 철학과 규칙을 보존하는 문서 저장소로 읽는다.

## 작업 전 해석 순서

1. `README.md`를 읽고 저장소 정체성을 이해한다.
2. `ugc-rules.yaml`을 읽고 고정 규칙을 확인한다.
3. `EXPORT_RULES.md`를 읽고 미러 공유 원칙을 확인한다.
4. `SIMULACRA_UGC_PLATFORM.md`를 읽고 철학 문맥을 보강한다.
5. 장기 비전이나 세계관 판단이 필요하면 `C:\Users\rneet\OneDrive\Desktop\DIS\EM\WHITEPAPER.md`와 `C:\Users\rneet\OneDrive\Desktop\DIS\EM\WORLD_LORE.md`를 참고한다.
6. 연속성이나 구현 상태 판단이 필요하면 `C:\Users\rneet\OneDrive\Desktop\DIS\EM\CONTINUITY_CONTRACT.md`와 `C:\Users\rneet\OneDrive\Desktop\DIS\EM\IMPLEMENTATION_CONTRACT.md`를 참고한다.

## 반드시 지킬 것

- `시뮬라크`를 개별 미니게임으로 축소 해석하지 말 것
- `공`을 시뮬라크 바깥의 별도 로비로 해석하지 말 것
- `포탈`을 본체와 동급으로 해석하지 말 것
- `오브젝트`를 단순 장식으로 해석하지 말 것
- `UGC`를 "배치 기능 있음" 수준으로 축소하지 말 것
- `시뮬라크`의 1차 제품 정체성을 3D UGC 창작 도시로 해석할 것
- 초기 코어 루프를 `배치 -> 저장 -> 전시`로 유지할 것
- 창작/검증/정착을 `작업실 -> 실험 월드 -> 메인 월드` 계층으로 이해할 것
- 저장 기본 단위를 개별 오브젝트가 아니라 `작품(creation/work)`으로 볼 것
- 오브젝트, 엔티티, AI 모듈을 분리해서 해석할 것
- 한국어 기준 문서를 유지할 것

## 수정 원칙

- 기존 철학 문장을 함부로 삭제하지 말고, 필요한 경우 보강 문장을 추가한다.
- 개념 충돌이 생기면 새 표현보다 기존 핵심 정의를 우선 보존한다.
- 구현 추정으로 문서를 덮어쓰지 않는다.
- 요청받지 않은 대규모 재구성은 하지 않는다.
- 다른 저장소로 내보내는 미러 규칙은 `EXPORT_RULES.md`를 따른다.

## 에이전트 판단 기준

문서를 읽고 판단할 때 아래 구분을 유지한다.

- 본체 세계: 시뮬라크
- 시작 구역: 공
- 교체 모듈: 포탈 기반 콘텐츠
- 지속 자산: 유저 흔적, 오브젝트, 창작 기록
- 초기 본체 루프: 배치, 저장, 전시
- 운영 계층: 작업실, 실험 월드, 메인 월드
- 장기 철학 기준: `DIS\EM`의 백서와 세계관 문서
- 연속성 기준: `DIS\EM`의 연속성 계약 문서
- 구현 상태 구분 기준: `DIS\EM`의 실구현 계약 문서
